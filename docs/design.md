# 仕様

## 主目的

- AIエージェントから簡単に呼び出せて、プロンプトを作ったらAIエージェントにすぐに渡せる
- AIエージェントに入力中のプロンプトを誤って渡さないように

## 機能一覧

- グローバルショートカットキーでアプリを開ける
  - 初期のショートカットキーは「Command, Command」と２連打（２回目のキーを離した際に、それまでに他のキー入力がなかった場合のみ発火）
  - その他、設定で修飾キーの２連打を含め、好きなショートカットキーに変更できる
  - 設定画面のHotkeyには「⇧Shift, ⇧Shift」のように修飾キーアイコンとキー名を併記する
  - HotkeyのCustomではポップアップ上で修飾キーと文字キーを同時に押して仮候補を入力し、Saveで反映、Cancelで現在の設定を維持する（CustomのデフォルトはShift+Control+G）
  - Custom Hotkeyの仮候補はアクセントカラーで表示し、仮候補がない場合または現在のCustom Hotkeyと同じ場合はSaveを無効にする
  - ホットキーでアプリを開いた際、現在のプロンプトが未保存（履歴にない）かつ空でなければ履歴に保存し、エディタの内容をクリアして新規入力状態にする
  - ホットキーで開いた際は常に「Current Prompt」が選択された状態になる
  - 履歴から過去のプロンプトを選択しても、「Current Prompt」に戻れば入力中の内容が保持されている
  - TemplateやReserveを編集中に「Current Prompt」に切り替えた際、TemplateやReserveに未保存の変更があれば自動的に履歴に保存する
  - アプリがすでにフォアグラウンドにある状態でホットキーが押された場合、Submit Targetのアプリをアクティブにして戻る
- アプリを開いたらテキストエディタにフォーカスがあたりすぐに文字入力できる
- ナビゲーションバーには以下のボタン（括弧内はショートカットキー）
  - Global Search (Command+F)
    - Template、Reserve、履歴を横断検索する
- ツールバーには以下のボタン（括弧内はショートカットキー）
  - Submit (Command+S)
    - このアプリに遷移する前のアプリに戻ってプロンプトの内容をペーストする
    - 遷移する前のアプリがわからなければDisabled
  - Copy (Command+C)
    - クリップボードにテキストエディタの全文をコピーする
    - ただしテキストエディタでテキストが選択されていたらそちらのコピーを優先
      - そのときはDisabled
- SubmitもしくはCopyを実行したときはそれぞれのボタンに0.5秒ProgressViewを表示して実行中であることを示す
- SubmitもしくはCopyを実行したらそのPromptを履歴として保存する
  - テキストエディタの内容が空、もしくはスペースや改行のみの場合は履歴に保存しない
  - 実行後、現在の選択が「Current Prompt」であればエディタの内容をクリアする
  - 履歴は左ペインのListに新しい順で並べる
  - 履歴はmacOS標準のスワイプUIで削除可能にする
  - 履歴の編集モード（historyEditingMode）が有効な場合、各セルの右端にゴミ箱アイコンの削除ボタンを表示し、即時削除を可能にする
  - 履歴はファイル保存してアプリを終了しても残るようにする
- テキストエディタはVimキーバインドでの入力に切り替え可能
  - テキストエディタ部分はWKWebView+CodeMirror6で実装
  - CodeMirror6のVim modeを使うため
- Command+Fでグローバル検索パレットを開ける
  - 入力中のキーワードでTemplate、Reserve、履歴の候補をリアルタイムに絞り込む
  - Enterで選択中の候補をPromptとしてCurrent Promptに適用する。履歴の場合はその履歴を選択する
  - 日本語などの文字入力の変換確定中のEnterは候補選択として扱わない
  - 入力欄にフォーカスがある状態でも上下キーまたはCommand+P/Command+Nで候補選択を移動できる
- Command+TでTemplate検索パレットを開ける
  - 入力中のキーワードでTemplate候補をリアルタイムに絞り込む
  - 候補はTemplate nameの前方一致、Template nameの部分一致、本文の部分一致、更新日時が新しい順で優先表示する
  - Enterで選択中の候補をPromptとしてCurrent Promptに適用する。候補がない場合はEnterでは何もしない
  - 日本語などの文字入力の変換確定中のEnterは候補選択として扱わない
  - 入力欄にフォーカスがある状態でも上下キーまたはCommand+P/Command+Nで候補選択を移動できる
- Command+Pで左ペインのListの1つ前の項目を選択・表示する。一番上の項目なら一番下の項目に移動する
- Command+Nで左ペインのListの次の項目を選択・表示する。一番下の項目なら「Current Prompt」に戻る
- Command+Hで最新の履歴を選択しエディタにフォーカスする。すでに履歴が選択されている場合は1つ前の（より古い）履歴に移動する
  - 標準の「新規ウィンドウ (Command+N)」は無効化する

## UI

- アプリの画面は２ペインで左にList、右がテキストエディタ
  - 左ペインには過去のプロンプト履歴が表示される
- 左ペインは表示/非表示の切り替えができる
  - macOS標準のSplitViewで実装する
- 画面下部に幾つかの機能をもったツールバー
- ショートカットキーが割り当てられているボタンは、マウスカーソルを合わせたときにショートカットキーだけの独自Hint Tipを即時表示する
  - Hint Tipはデフォルトでは対象Viewの右隣に表示し、必要に応じて左隣、上、下、または小さめの表示に変更できる
- メニューバーも具備
  - シンプルな設定画面（Command+,）
  - macOSの標準コンポーネントを使った設定画面
  - 設定には以下がある
    - Hotkey（このアプリを起動するショートカットキー）
    - Keyboard Shortcuts
      - Hotkey以外のアプリ内ショートカットキーを変更できる
      - GeneralのHotkeyの下からドリルダウンして設定画面に遷移する
      - 各ショートカット項目を押すとHotkeyのCustomと同じ形式のポップアップでキー入力を録画できる
      - ポップアップのSaveはキーボードショートカット設定画面内のドラフトにのみ反映し、アプリ全体にはまだ反映しない
      - キーボードショートカット設定画面のSaveで変更済みショートカットをまとめてアプリに反映する
      - 変更が1つもない場合はSaveをDisabledにする
      - 未保存の変更がある状態でGeneralへ戻ろうとした場合はconfirmationDialogで破棄確認する
    - テキストエディタ設定
      - Vimキーバインドへの切り替えのOn/Off（デフォルトはOff）
      - 行の折り返しのOn/Off（デフォルトはOn、日本語の長文も画面幅に合わせて折り返す）
    - 履歴設定
      - 保存件数の上限（10〜1000件、デフォルト100件）
      - 履歴のクリアボタン（要確認ダイアログ）
